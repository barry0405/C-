命令模式
==
#   命令模式 将“请求”封装成对象，以便使不同的请求、队列或者日志来参数化其他对象。命令模式也支持可撤销的操作。 当需要将发出请求的对象和执行请求的对象解耦的时候，使用命令模式。
##   一个命令对象通过在特定接收者上绑定一组动作来封装一个请求。要达到这一点，命令对象将动作和接收者包进对象中。这个对象只暴露出一个execute()方法，当此方法调用的时候，接收者就会进行这些动作。从外面来看，其他对象不知道究竟哪个接收者进行了那些动作，只知道如果调用execute()方法，请求的目的就达到。

要点
=
# 命令模式将发出请求的对象和执行请求的对象解耦。
# 在被解耦的两者之间是通过命令对象进行沟通的。命令对象封装了接收者和一个或一组动作
# 调用者通过命令对象的execute()发出请求，这会使得接收者的动作被调用
